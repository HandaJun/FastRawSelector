## 1. 목적

큰 기능 로드맵(A~J)은 채워진 상태다.  
이 문서는 **A~J 이후 잔여 기술 부채·품질·배포**를 **파트 단위**로 끊어서 진행하기 위한 실행 계획이다.

| 원칙 | 내용 |
|---|---|
| **한 파트 = 한 목표** | 수락 기준 통과 후 다음 파트 |
| **업데이트는 나중** | 앱 내 자동 업데이트 체크(D-5)는 **후순위·보류** |
| **배포는 GitHub** | 릴리즈 zip을 GitHub Releases에 올리고, 사용자는 거기서 받아 씀 |
| **범위 밖** | 전면 MVVM, .NET 마이그레이션, Linux/macOS, 셸 썸네일 확장 |

## 12. 검증 체크리스트 (공통)

매 파트 종료 시:

1. 솔루션 빌드 (Debug 또는 Release)  
2. 해당 파트 수락 기준 수동 확인  
3. `%AppData%\Roaming\FastRawSelector\logs\`  
4. 회귀 스모크: 폴더 열기 → 선택 → 이동 → 그리드 → 설정 → 종료 → 재실행  
5. 이 문서 ✅ 표시
